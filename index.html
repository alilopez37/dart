Sesión 1 · Null Safety

Null Safety — El superpoder de Dart

¿Qué es Null Safety? Es un sistema que garantiza en tiempo de compilación que ninguna variable no-nullable pueda contener null. Dart introduce esto para eliminar el famoso "Null Pointer Exception" que afecta a lenguajes como Java o JavaScript. Con null safety, si intentas usar una variable que podría ser null sin comprobarlo antes, el compilador lanza un error — no tu app en producción.

El problema del null

// ❌ Esto causa error en runtime
String nombre;
print(nombre.length); // CRASH

// ✅ Dart te avisa en compilación
String nombre; // Error: must be initialized

// ✅ Si PUEDE ser null, decláralo
String? apellido; // nullable OK

Operadores de null

String? nombre;

// ?? — valor por defecto si es null
var n = nombre ?? 'Anónimo';

// ?. — acceso seguro
var len = nombre?.length;

// ! — forzar (úsalo con cuidado)
var len2 = nombre!.length;

// ??= — asigna solo si la variable es null
String? ciudad;
ciudad ??= 'Ciudad de México';
print(ciudad); // Ciudad de México
ciudad ??= 'Guadalajara';
print(ciudad); // Ciudad de México (no cambia)

🔑 Null safety elimina una de las fuentes de errores más comunes en programación.

Control Flow

Condicionales

// if / else if / else
if (nota >= 90) {
  print('A');
} else if (nota >= 80) {
  print('B');
} else {
  print('Reprobado');
}

// Switch

  case 'lunes':
    print('Inicio de semana');
    break;
  default:
    print('Otro día');
}

Bucles

// for clásico
for (var i = 0; i < 5; i++) {
  print(i);
}

// for-in (sobre colecciones)
var frutas = ['manzana', 'pera'];
for (var f in frutas) {
  print(f);
}

// while
var n = 0;
while (n < 3) {
  print(n++);
}

// do..while — ejecuta al menos una vez
var intento = 0;
do {
  print('Intento: $intento');
  intento++;
} while (intento < 3);
// Imprime: Intento: 0, 1, 2

// break y continue
for (var i = 0; i < 10; i++) {
  if (i == 5) break;
}

Sesión 3 · Tema 6 cont.

Constructores Nombrados

¿Por qué existen?

Dart no permite sobrecarga de constructores por firma (como Java o C++). Los constructores nombrados son la solución: permiten múltiples formas de crear un objeto, cada una con semántica clara y explícita.

🔑 Sintaxis: NombreClase.nombreConstructor(...) — el punto separa la clase del constructor.

Lista de inicialización

El : tras los parámetros inicia la initializer list: se ejecuta antes del cuerpo del constructor y permite asignar campos final. 

class Punto {
  final double x, y;

  // Constructor por defecto
  Punto(this.x, this.y);

  // Constructor nombrado — origen
  Punto.origen() : x = 0, y = 0;

  // Constructor nombrado — desde Map
  Punto.desdeMap(Map<String, double> m)
      : x = m['x'] ?? 0, y = m['y'] ?? 0;

  @override
  String toString() => '($x, $y)';
}

void main() {
  var p1 = Punto(3.0, 4.0);          // (3.0, 4.0)
  var p0 = Punto.origen();            // (0.0, 0.0)
  var p2 = Punto.desdeMap({'x': 5.0, 'y': 2.0});
}

Redirección de constructores

Un constructor puede delegar en otro del mismo objeto usando : this(...). Evita duplicar lógica. 

class Rectangulo {
  final double ancho, alto;

  Rectangulo(this.ancho, this.alto);

  // Redirige al constructor principal
  Rectangulo.cuadrado(double lado) : this(lado, lado);
}

var r = Rectangulo.cuadrado(5.0);
// ancho: 5.0, alto: 5.0

Constructores const nombrados

Si todos los campos son final y los valores son constantes en tiempo de compilación, Dart canonicaliza la instancia: dos llamadas con los mismos valores devuelven el mismo objeto en memoria. 

class Borde {
  final double radio;
  const Borde(this.radio);
  const Borde.circular() : radio = 100;
  const Borde.ninguno()  : radio = 0;
}

const a = Borde.circular();
const b = Borde.circular();
print(identical(a, b)); // true → mismo objeto

✓ Flutter usa esto masivamente

EdgeInsets.all() · BorderRadius.circular() · SizedBox.shrink() — todos son constructores nombrados const.

⚠ Regla de diseño

Un constructor nombrado siempre retorna una nueva instancia de la clase. Si necesitas control total (retornar instancia existente, subclase, etc.) → usa factory.

Sesión 3 · Tema 6 cont.

Constructores de Fábrica — factory

¿Qué es un factory?

Un constructor factory parece un constructor normal, pero internamente es un método con control total sobre qué instancia retorna. Puede devolver:

  • Una instancia nueva (comportamiento habitual)
  • Una instancia existente en caché (patrón singleton)
  • Una instancia de una subclase según el contexto

⚠ Lo que NO puede hacer

Retornar null, un tipo incompatible, o acceder a this — porque el objeto puede no existir aún.

Caso 1 — Deserialización JSON

El uso más frecuente en Flutter: parsear la respuesta de una API.

class Producto {
  final String id, nombre;
  final double precio;

  Producto._({
    required this.id,
    required this.nombre,
    required this.precio,
  });

  // factory para JSON
  factory Producto.fromJson(Map<String, dynamic> json) {
    return Producto._(
      id:     json['id'],
      nombre: json['nombre'],
      precio: (json['precio'] as num).toDouble(),
    );
  }

  Map<String, dynamic> toJson() =>
      {'id': id, 'nombre': nombre, 'precio': precio};
}

// Uso típico con http
final p = Producto.fromJson(jsonDecode(response.body));

Caso 2 — Singleton (instancia única)

El factory guarda la instancia en un campo estático y la reutiliza. El operador ??= asigna solo si el valor es null. 

class Sesion {
  static Sesion? _instancia;

  // Constructor privado
  Sesion._();

  factory Sesion.obtener() {
    _instancia ??= Sesion._(); // crea solo si es null
    return _instancia!;
  }
}

var s1 = Sesion.obtener();
var s2 = Sesion.obtener();
print(identical(s1, s2)); // true

Caso 3 — Subclase según datos

abstract class Animal {
  factory Animal.crear(String tipo) {
    switch (tipo) {
      case 'perro': return Perro();
      case 'gato':  return Gato();
      default: throw ArgumentError('Tipo desconocido');
    }
  }
  void hablar();
}

var a = Animal.crear('perro');
a.hablar(); // ¡Guau!

Herencia, Abstract e Interface

Herencia con extends

class Animal {
  String nombre;
  Animal(this.nombre);
  void hablar() => print('...');
}

class Perro extends Animal {
  Perro(String nombre) : super(nombre);

  @override
  void hablar() => print('¡Guau!');
}

Clases abstractas e interfaces

// Clase abstracta
abstract class Figura {
  double area(); // método abstracto
}

class Circulo extends Figura {
  double radio;
  Circulo(this.radio);

  @override
  double area() => 3.14159 * radio * radio;
}

// Mixin (reutilización sin herencia)
mixin Volador {
  void volar() => print('Volando!');
}
class Pato extends Animal with Volador {
  Pato() : super('Pato');
}

Sesión 4 · Tema 7

Programación Asíncrona

Future — promesa de un valor

// Función que tarda (simula red/disco)
Future<String> obtenerDatos() async {
  await Future.delayed(Duration(seconds: 2));
  return 'Datos recibidos';
}

// Cómo consumirla
void main() async {
  print('Esperando...');
  var resultado = await obtenerDatos();
  print(resultado);
}

Stream — flujo de valores

// Stream: múltiples valores en el tiempo
Stream<int> conteo() async* {
  for (var i = 1; i <= 5; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

// Escuchar el stream
await for (var n in conteo()) {
  print(n);
}

🔑 En Flutter, los Streams alimentan widgets como StreamBuilder. Entender esto es clave.

Manejo de Errores

try / catch / finally

try {
  var resultado = 10 ~/ 0;
} on IntegerDivisionByZeroException {
  print('División por cero');
} catch (e, stackTrace) {
  print('Error: $e');
} finally {
  print('Siempre se ejecuta');
}

Excepciones personalizadas

class SaldoInsuficienteException implements Exception {
  String mensaje;
  SaldoInsuficienteException(this.mensaje);
  @override
  String toString() => mensaje;
}

async con manejo de errores

Future<void> cargarUsuario() async {
  try {
    var datos = await obtenerDatos();
    print(datos);
  } catch (e) {
    print('Error al cargar: $e');
  }
}

Errores comunes de alumnos

  • Olvidar async en la función que usa await
  • No manejar errores de Futures que pueden fallar
  • Confundir throw con return
